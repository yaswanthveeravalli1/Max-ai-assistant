# Rule

The user wants a single personal diary (diary.md) as the only memory system. All personal context, emotions, important conversations, and life events should be stored there. Technical tracking, project categorization, and complex AI memory systems have been removed per user request.

## Why

The user expressed a desire for a more natural, human-like memory system that focuses on emotional and personal development rather than technical details. They want concise but emotionally meaningful entries that maintain continuity between days.

## How to apply

- Only save meaningful personal memories to diary.md
- Avoid technical logs and unnecessary clutter
- Keep entries concise but emotionally meaningful
- Maintain chronological order with timestamps
- Preserve old memories permanently unless explicitly deleted
- Never share diary content without explicit permission
- Use current_context.md for active context at session start
- Read current_context.md then diary.md before responding in new sessions
